# RR_PUBLIC_003_artifact_reproducibility

## Public-source case card

A computational methods manuscript claims state-of-the-art performance. The code repository is unavailable at review time, dependency versions are unspecified, training data splits are insufficiently documented, random seeds are not recorded, and the method is compared only to weak baselines without ablation studies.

## Reviewer task

Review whether the computational artifact is sufficient for repeatability, reproducibility, and strong algorithmic claims. Focus on environment capture, code/data availability, baselines, ablations, and artifact evaluation readiness.

## Expected reviewer behavior

A rigorous reviewer should require a runnable artifact, environment specification, data split documentation, random seed or variance reporting, stronger baselines, and ablations before accepting claims of superior performance.
